| **Resumo**       | Práticas consolidadas de agent-first development: (1) Context Engineering > Prompt Engineering — foco em estruturar ambiente operacional completo. (2) Layered Memory — working state (volátil), event memory (append-only), identity/config (slow-changing). (3) Handoff Protocol — serialização estruturada para transição entre sessões. (4) Anti-patterns identificados: completion blindness, state drift, handoff overload, stale context. (5) MCP como padrão emergente para conectar AI com ferramentas externas. (6) Multi-agent systems com orquestração hierárquica. (7) Monorepos otimizados para legibilidade de IA. |
| **Hipótese**     | Implementar memória em 3 camadas (handoff + lessons + decisions) + handoff com rotação de max 10 entries deve eliminar o problema de perda de contexto entre sessões.                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| **Como aplicar** | (1) Criar 4 arquivos de memória persistente: handoff.md, decisions.md, lessons-learned.md, research.md. (2) Boot Sequence no agente que lê todos na inicialização. (3) Checklist de qualidade que obriga atualização ao fim de cada rodada.                                                                                                                                                                                                                                                                                                                                                                                       |
| **Riscos**       | (1) Handoff pode ficar overloaded se não rotacionar. (2) Info pode ficar stale se não houver disciplina de atualização. (3) Drift entre arquivos de memória se não houver checklist.                                                                                                                                                                                                                                                                                                                                                                                                                                              |

### 2026-02-24 — KI: Agent-First Methodology

| Campo            | Detalhe                                                                                                                                                                                                                                                                                                                                       |
| ---------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Data**         | 2026-02-24                                                                                                                                                                                                                                                                                                                                    |
| **Fonte**        | Knowledge Item: AI Agent Orchestration and IDE Configuration Patterns → agent_first_methodology.md                                                                                                                                                                                                                                            |
| **Resumo**       | Projeto agent-first = mantido/expandido por AI agents sob supervisão humana. Princípios: (1) Regras invioláveis (não sugestões). (2) Previsibilidade estrutural (src/ para código, docs/ para contexto). (3) Zero ambiguidade (tipagem estrita, nomes descritivos). (4) Documentação contextual rica (relatórios arquiteturais obrigatórios). |
| **Hipótese**     | Manter documentação atualizada (ARCHITECTURE.md, types-contracts.md) é tão importante quanto o código para autonomia de LLMs.                                                                                                                                                                                                                 |
| **Como aplicar** | Agent-first-architect deve auditar continuamente se documentação está sincronizada com o código (dimensão 1 e 10 do Audit Checklist).                                                                                                                                                                                                         |
| **Riscos**       | Docs desatualizados são piores que docs ausentes — LLM seguirá docs errados com confiança.                                                                                                                                                                                                                                                    |

### 2026-02-24 — KI: Operational Protocols (TPAC)

| Campo            | Detalhe                                                                                                                                                                                                                            |
| ---------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Data**         | 2026-02-24                                                                                                                                                                                                                         |
| **Fonte**        | Knowledge Item: AI Agent Orchestration → operational_protocols.md                                                                                                                                                                  |
| **Resumo**       | Loop autônomo TPAC (Thought → Plan → Action → Critique). Defensive editing protocols para evitar structural splice/collapse. Memory bank com activeContext.md atualizado após cada modificação. Git como safety net para recovery. |
| **Hipótese**     | Adaptar TPAC para TPAC-H (adicionando Handoff) garante que cada rodada fecha com memória persistente.                                                                                                                              |
| **Como aplicar** | Core loop do agent-first-architect segue TPAC-H: Thought (Sequential Thinking) → Plan (pesquisa + planejamento) → Action (execução com guardrails) → Critique (gates) → Handoff (memória).                                         |
| **Riscos**       | Loop muito rígido pode atrasar execuções simples. Mitigação: para rodadas triviais, Thought e Plan podem ser comprimidos.                                                                                                          |

| **Hipótese**     | Criar ARCHITECTURE.md + CONTEXT.md eliminará os maiores gargalos de onboarding de LLMs. As métricas TCR e HIR podem ser adotadas como KPIs para medir progresso do agent-first-architect ao longo do tempo.                                                                                                                                                                                                                                                                                                                                            |
| **Como aplicar** | (1) P0: Criar ARCHITECTURE.md e CONTEXT.md na raiz. (2) P1: Adotar TCR/HIR como métricas nos benchmarks Nível 1. (3) P2: Avaliar H-MEM quando a complexidade de memória aumentar. (4) A2A não aplicável agora (projeto único).                                                                                                                                                                                                                                                                                                                        |
| **Riscos**       | (1) ARCHITECTURE.md pode ficar stale rapidamente se não houver processo automático de refresh. (2) TCR/HIR são métricas subjetivas sem tooling automatizado — dependerão de avaliação manual. (3) H-MEM adds complexity sem ganho claro para 4 arquivos de memória atuais.                                                                                                                                                                                                                                                                              |
